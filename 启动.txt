================================================================================
                        AI 小说创作助手 - 启动说明
================================================================================

项目包含三个独立模块：
  - Python 后端（FastAPI）   → python-backend/    主要后端，负责 AI 创作
  - Java 后端（Spring Boot） → pom.xml            可选，文章创作器功能
  - 前端（Vue 3）            → frontend/          Web 界面


================================================================================
                            环境要求
================================================================================

  Python       3.10+
  Node.js      18+
  MySQL        8.0+
  Redis        7.x
  Java         21+       （仅 Java 后端需要）
  Docker       24+       （仅 Docker 方式需要）


================================================================================
                        方式一：Docker Compose 启动（推荐）
================================================================================

适合快速部署，一键启动 MySQL + Redis + 后端 + 前端。

  1. 进入项目目录
     cd ai-novel

  2. 配置环境变量
     cp python-backend/.env.example python-backend/.env
     编辑 python-backend/.env，填写以下内容：

     必填：
       DASHSCOPE_API_KEY=你的API Key
       DASHSCOPE_MODEL=MiMo-V2.5-Pro
       DASHSCOPE_BASE_URL=https://token-plan-cn.xiaomimimo.com/v1

     可选（有默认值，按需修改）：
       MYSQL_ROOT_PASSWORD=123456          # MySQL root 密码
       MYSQL_DATABASE=ai_passage_creator   # 数据库名
       SESSION_SECRET_KEY=任意随机字符串    # Session 密钥
       PEXELS_API_KEY=                     # Pexels 图片搜索 Key

  3. 启动所有服务
     docker compose up -d --build

     首次启动需要拉取镜像和构建，耐心等待。

  4. 验证服务状态
     docker compose ps
     确保 mysql、redis、backend、frontend 都是 running 状态。

  5. 访问
     前端：     http://localhost
     后端 API： http://localhost:8123
     API 文档： http://localhost:8123/docs

  常用命令：
     docker compose up -d          # 启动
     docker compose down            # 停止
     docker compose logs -f backend # 查看后端日志
     docker compose restart backend # 重启后端
     docker compose up -d --build   # 重新构建并启动


================================================================================
                        方式二：手动启动（开发调试用）
================================================================================

适合本地开发，方便修改代码和调试。

-------------------- 第 1 步：启动 MySQL 和 Redis --------------------

  方法 A：用 Docker 只启动数据库（推荐）
    docker compose up -d mysql redis

  方法 B：本地已安装 MySQL 和 Redis
    直接启动服务即可，确保 MySQL 在 3306 端口，Redis 在 6379 端口。

  验证：
    docker compose ps              # Docker 方式
    mysql -uroot -p -e "SELECT 1"  # 测试 MySQL 连接
    redis-cli ping                 # 测试 Redis 连接，应返回 PONG

-------------------- 第 2 步：初始化数据库（仅首次） --------------------

  SQL 脚本使用 CREATE TABLE IF NOT EXISTS，重复执行不会丢数据。

  方法 A：命令行执行
    mysql -uroot -p < sql/init_database.sql

  方法 B：登录 MySQL 手动执行
    mysql -uroot -p

    CREATE DATABASE IF NOT EXISTS ai_passage_creator DEFAULT CHARSET utf8mb4;
    USE ai_passage_creator;

    SOURCE sql/create_table.sql;
    SOURCE sql/create_novel_tables.sql;
    SOURCE sql/add_phase_fields.sql;
    SOURCE sql/add_article_style.sql;
    SOURCE sql/add_vip_payment.sql;
    SOURCE sql/update_quota.sql;

  验证：
    mysql -uroot -p -e "USE ai_passage_creator; SHOW TABLES;"
    应看到 user、novel、chapter、character、foreshadowing 等表。

-------------------- 第 3 步：配置 Python 后端环境变量 --------------------

  cd python-backend
  cp .env.example .env

  编辑 .env，必须填写的字段：

    # 数据库（和你本地 MySQL 配置一致）
    DB_HOST=localhost
    DB_PORT=3306
    DB_NAME=ai_passage_creator
    DB_USER=root
    DB_PASSWORD=你的MySQL密码

    # Redis
    REDIS_HOST=localhost
    REDIS_PORT=6379
    REDIS_PASSWORD=

    # AI 模型（必填）
    DASHSCOPE_API_KEY=你的API Key
    DASHSCOPE_MODEL=MiMo-V2.5-Pro
    DASHSCOPE_BASE_URL=https://token-plan-cn.xiaomimimo.com/v1

    # Session（随便填一个随机字符串）
    SESSION_SECRET_KEY=your-random-secret-key

  其他字段有默认值，一般不用改。

-------------------- 第 4 步：启动 Python 后端 --------------------

  cd python-backend

  # 创建并激活虚拟环境（任选一种）
  conda create -n novel python=3.10 && conda activate novel
  # 或
  python -m venv venv && source venv/bin/activate     # Linux / Mac
  python -m venv venv && venv\Scripts\activate        # Windows

  # 安装依赖（首次需要，之后跳过）
  pip install -r requirements.txt

  # 启动后端
  python -m uvicorn app.main:app --host 0.0.0.0 --port 8567

  验证：
    浏览器打开 http://localhost:8567/docs 能看到 API 文档说明启动成功。

-------------------- 第 5 步：启动前端 --------------------

  另开一个终端：

  cd frontend

  # 安装依赖（首次需要，之后跳过）
  npm install

  # 启动开发服务器
  npm run dev

  验证：
    浏览器打开 http://localhost:5173

-------------------- 第 6 步：登录 --------------------

  账号：admin
  密码：12345678


================================================================================
                        再次启动（以前运行过的情况）
================================================================================

如果之前已经配置好环境和数据库，不需要重新初始化，直接启动即可。

  Docker 方式：
    cd ai-novel
    docker compose up -d

  手动方式：
    1. 确保 MySQL 和 Redis 在运行
       docker compose up -d mysql redis

    2. 启动后端（另开终端）
       cd python-backend
       conda activate novel
       python -m uvicorn app.main:app --host 0.0.0.0 --port 8567

    3. 启动前端（另开终端）
       cd frontend
       npm run dev

  注意事项：
    - 数据库不用重新初始化（SQL 脚本用 IF NOT EXISTS，不会覆盖已有数据）
    - 依赖不用重新安装（除非拉取了新代码导致依赖变化）
    - .env 不用重新修改（除非要换 API Key 或模型）
    - 如果依赖有变化：pip install -r requirements.txt 或 npm install


================================================================================
                        方式三：启动 Java 后端（可选）
================================================================================

项目根目录有 pom.xml，是 Spring Boot 后端，用于文章创作器功能。
和 Python 后端是独立的，根据需要选择启动哪个。

  # 确保 Java 21+ 已安装
  java -version

  # 用 Maven 启动
  mvn spring-boot:run

  # 或打包后运行
  mvn clean package -DskipTests
  java -jar target/ai-passage-creator-0.0.1-SNAPSHOT.jar

================================================================================
                            常见问题
================================================================================

Q: 启动报错 "Access denied for user"
A: 检查 python-backend/.env 中 DB_USER 和 DB_PASSWORD 是否和 MySQL 一致。
   Docker 方式下密码默认是 123456，对应 docker-compose.yml 中的 MYSQL_ROOT_PASSWORD。

Q: AI 调用失败 / 超时
A: 检查三项：
   1. DASHSCOPE_API_KEY 是否有效
   2. DASHSCOPE_BASE_URL 是否可访问（浏览器打开看是否有响应）
   3. DASHSCOPE_MODEL 模型名称是否正确

Q: 前端请求后端 404
A: 确认后端已启动，浏览器打开 http://localhost:8567/docs 验证。
   如果后端正常，检查 frontend/vite.config.ts 中的代理配置。

Q: Docker 启动后数据库表不存在
A: Docker MySQL 只在首次创建容器时执行 initdb.d 中的脚本。
   如果之前已经创建过容器，需要先清除数据卷：
     docker compose down -v    # 删除容器和数据卷
     docker compose up -d      # 重新创建

Q: 重新执行 SQL 脚本会丢数据吗
A: 不会。脚本用的是 CREATE TABLE IF NOT EXISTS，已存在的表会被跳过。

Q: 修改 .env 后不生效
A: .env 只在服务启动时读取，修改后必须重启：
   手动方式：Ctrl+C 停止后端，重新运行 uvicorn 命令
   Docker 方式：docker compose restart backend

Q: 端口被占用
A: 检查端口占用情况：
   Windows: netstat -ano | findstr :8567
   Linux/Mac: lsof -i :8567
   然后修改 .env 中的 SERVER_PORT 或 docker-compose.yml 中的端口映射。

Q: pip install 报错
A: 确保使用 Python 3.10+：python --version
   建议使用虚拟环境，避免依赖冲突。

Q: npm install 报错
A: 确保 Node.js 18+：node --version
   尝试清除缓存：npm cache clean --force，然后重新 npm install
